# 大觀國中暑期排課系統

這是依據「大觀國中暑期課表排課系統 Codex 可執行開發任務書」建立的 MVP。前端可直接放在 GitHub Pages，排課演算法在瀏覽器端執行，Google Apps Script 只負責讀寫 Google Sheets。

## 目前完成

- 管理者 / 教師登入，未設定 GAS 時使用內建示範資料。
- 國二 3 週、國三 5 週，每日 4 節，以第 1-2 節與第 3-4 節為固定連堂區塊。
- 班級課表與教師課表檢視。
- 一鍵自動排課，保留鎖定與預排課程。
- 預排 / 鎖定編輯面板。
- 連堂拖拉調課，若產生衝突會退回。
- 調課、手動補排、清空課程、一鍵排課、載入版本後可復原上一步，也可重做。
- 教師撞課、班級同科連四、同班同科同老師、教師週次、可授課星期、不可排課日期、排課節次、授課班級、場地容量、社會科指定科目等檢查。
- 衝突檢查項目會依教師、班級、教師節數、節數不足 / 未排入、空白課表、場地、社會科等類型使用不同色塊，可依類型篩選，也可逐筆或依目前篩選批次刪去；重新檢查時會再次依目前課表重新產生。
- 社會科自動 / 手動安排。
- 版本歷史，本機可儲存與載入；設定 GAS 後可寫入 Sheets。
- 班級、教師、總課表 CSV 匯出；PDF 可用瀏覽器列印，也支援批次列印全部班級或全部教師課表。
- 班級 / 教師課表可選單週或一次檢視全部週次；PDF 輸出會依年級週數列出課表，並顯示週次日期，例如 `第 1 週 7/13~7/17`。
- 規則式最佳化建議，預留後續 AI API 串接位置。

## 快速使用

直接開啟 `index.html` 即可測試示範資料。部署到 GitHub Pages 時，將整個資料夾推到 repository，並啟用 Pages。

若上傳 GitHub Pages 後看不到右上角「版本 20260619-3」，代表 Pages 仍讀到舊版 `index.html` 或檔案沒有覆蓋到正確位置。請確認 `index.html` 在 repository 根目錄，`js/app.js`、`js/constraints.js`、`js/scheduler.js` 在 `js` 資料夾內，再等 GitHub Pages 完成部署後強制重新整理。

手動預排或調課時，系統會先檢查整張課表是否仍有硬性衝突。若錯誤訊息顯示「目前課表另有硬性衝突」，代表正在編輯的格子不一定錯，而是其他班級或教師設定已有必須先處理的問題。

調課模式下，滑鼠移到格子時只會顯示與該堂課或目標格有關的限制；若整張課表另有無關硬性衝突，格子提示會引導先到「衝突檢查」處理，避免把其他班級的錯誤誤認為目前格子的問題。

教師課表檢視會依「教師姓名」合併顯示。同一位老師若因不同科目分成多個教師代碼，例如 `T503 何中楷 理化` 與 `T504 何中楷 生物`，下拉選單會顯示為同一位「何中楷」，教師課表、批次列印與教師 CSV 會一起列出兩個教師代碼的課。

同姓名教師也會被視為同一位老師檢查衝堂。即使教師代碼不同，例如 `T503 何中楷 理化` 與 `T504 何中楷 生物`，系統也不會允許理化與生物排在同一週、同一天、同一連堂區塊。

| 角色 | Email | 密碼 |
|---|---|---|
| 管理者 | admin@daguan.ntpc.edu.tw | admin123 |
| 教師 | teacher@daguan.ntpc.edu.tw | teacher123 |

## 連接 Google Sheets

目前系統已內建大觀國中使用的 GAS Web App URL，登入時只需要輸入 Email 與密碼。

若未來要更換 GAS：

1. 依照 `gas/README_GAS_DEPLOY.md` 建立 Google Sheets 與部署 Apps Script Web App。
2. 打開系統後進入「資料設定」。
3. 貼上新的 GAS Web App URL 並儲存。
4. 系統會重新讀取 Sheets 資料。

若已登入後才修改 Google Sheets，可直接按主畫面上方的「重新讀取試算表」取得最新資料，不必登出重登。若修改的是班級、教師、課程節數或預排資料，重新讀取後請再按「一鍵自動排課」或重新檢查衝突。

GAS 的 `initializeSheets` 會建立繁體中文分頁與中文表頭，例如「使用者帳號」「教師設定」「班級設定」。前端會自動把中文表頭對應回程式欄位。

上課週數依年級設定，不需要在每個班級逐一填寫。請在「系統設定」中設定 `grade8Weeks=3`、`grade9Weeks=5`；班級設定只要填年級，系統會自動判斷 8 年級排 3 週、9 年級排 5 週。若舊表還留有「上課週次」欄位，請更新 `gas/Code.gs` 後執行 `action=initializeSheets&forceHeaders=true` 或 `action=localizeHeaders` 清理表頭。

「教師設定」不再使用「科目群組」欄位；老師可教哪些科目，請只填在「可授科目」。若一位老師可教多科，可填 `地理,歷史`。「授課班級」可填 `901,902` 這種逗號分隔清單；留空代表該教師可授課所有班級。「教師職位」可填 `組長`、`導師`、`專任`；留空會視為 `專任`。「可授課星期」可填 `1,2,4,5` 或 `星期一,星期二,星期四,星期五`；留空代表週一到週五都可排。「不可排課日期」可填 `7/15,7/22` 或 `2026-07-15,2026-07-22`；留空代表沒有特定日期限制。「排課節次」可填 `1,2,3,4` 或 `3,4`；留空代表第 1-4 節都可排。

「課程節數配額」的每一列「目標節數」就是該年級、該科目的排課目標。若 9 年級地理目標節數為 `10`、歷史目標節數為 `10`，系統會分別排足地理 10 節與歷史 10 節，不會合併計算。社會科可用兩種方式設定：若科目填 `社會`，系統會依「社會科安排」把總節數分配給指定的公民、歷史、地理；若科目直接填 `地理`、`歷史`、`公民`，系統會分科計算各自節數。

社會科某班要上哪兩科，以「社會科安排」為最後依據；若「課程節數配額」直接列出 `地理`、`歷史`、`公民`，系統只會替該班排入「社會科安排」中選到的科目。例如 9卓 設為 `公民、歷史`，就不會再因為同年級有 `地理` 配額而排入地理。

若「班級設定」的社會科模式為 `手動`，且「手動指定社會科」填入例如 `公民,歷史`，系統會以班級設定為準更新社會科安排，不會再被「社會科安排」分頁中的舊科目 A/B 覆蓋。

## 暑輔日期設定

週次日期由 `System_Settings` 的 `scheduleStartDate` 控制，請填第一週星期一日期。內建示範值為 `2026-07-13`，因此第 1 週會顯示 `7/13~7/17`。各年級上課週數由 `grade8Weeks`、`grade9Weeks` 控制，內建為 8 年級 3 週、9 年級 5 週。

## 排課規則設計藍圖

- 同一班、同一科目，必須由同一位老師授課。
- 班級的實際排課週數依年級設定決定；例如 9 年級依 `grade9Weeks=5` 排第 1-5 週，8 年級依 `grade8Weeks=3` 排第 1-3 週。
- 教師設定的「授課班級」會限制老師可被排入的班級；留空代表可授課所有班級。
- 教師設定的「教師職位」會影響自動排課集中度；`導師`、`專任` 會優先排成同一天 4 節，`組長` 不強制集中在同一天，方便保留行政彈性。
- 教師設定的「可授課星期」會限制老師可排入的星期；例如填 `1,2,4,5` 時，系統會避開星期三。
- 教師設定的「不可排課日期」會限制老師不可排入特定日期；例如填 `7/15,7/22` 時，系統會依暑輔起始年份避開這兩天，衝突檢查也會列出已排到不可排日期的課程。
- 教師設定的「排課節次」是硬性限制；例如填 `3,4` 時，該教師只能排進第 3-4 節連堂區塊，不會排入第 1-2 節。
- 自動排課會先依「班級＋科目」決定授課老師，再依老師「可授課週次」拆分每週課程。
- 若老師 1 到 5 週都可授課，系統會盡量把總節數平均分散到每週，例如國文 20 節會排成每週 4 節。
- 若授課老師只能上部分週次，系統會把節數壓縮到可授課週次內，例如 20 節且只能上第 1、2、3 週時，會優先排成第 1 週 8 節、第 2 週 8 節、第 3 週 4 節。
- 自動排課第一輪會避免同班同科同一天連上 4 節；若依所有正常規則仍無法排足該科目標節數，最後補排階段會允許同班同科同一天 4 節，並在課程上標記為系統為補足節數同意的例外。
- 衝突檢查會提醒教師單週授課負荷；只有同一位老師同一週超過「教師設定」的每週節數上限時，才會提供移到其他週、改派同科老師，或請主任徵詢增加授課老師的建議。
- 衝突檢查中的教師相關提醒會顯示「科目＋教師姓名」，例如「公民老師 羅思穎」，方便判斷要找哪一科協調。
- 若班級科目未排滿或自動排課有區塊未排入，衝突檢查會補上可能原因，例如無可授課教師、授課班級未包含該班、可授課週次不符、可授課星期不符、不可排課日期、排課節次不符、教師同時段已有課，或空白連堂區塊不足。
- 衝突檢查會依類型顯示不同色塊與標籤，例如班級、教師節數、節數不足 / 未排入、空白課表、場地、社會科；可用上方類型篩選，也可逐筆刪去或按「刪去目前篩選」批次刪去已處理或暫時忽略的提醒。按「檢查衝突」會重新產生完整清單，硬性衝突即使被刪去也不會允許儲存版本。
- 課程卡片會顯示「場地」與「來源」；「場地：一般教室」代表使用普通教室，「來源：系統排課」代表由一鍵自動排課產生。
- 管理者可按課程卡片上的「調課」，系統會展開五週課表並以顏色標示可移入、可交換、需確認、目前位置與不可調的位置；選定位置後會先顯示相關教師的調課後課表預覽，再按確認才套用。
- 同班同科同一天 4 節原則上列為硬性限制；自動排課第一輪與一般補救流程會避免排入，只有最後補排仍無法補足節數時才會由系統標記為同意例外。若手動調課確定無法避免，系統會顯示原因並要求管理者再次按「我了解，仍要調課」才會套用，並在課程上標記「已同意連四」。
- 課表調整會保留最近 30 次操作，可用「復原上一步」退回，也可用「重做」恢復。復原只影響目前畫面中的課表，確認無誤後仍需按「儲存版本」才會寫入版本。
- 預排與鎖定課程會優先保留，系統只會補足尚未達成的週次與節數。
